# 上海红色景点可视化 App

一个纯前端静态网页应用，用地图可视化上海红色景点（含四行仓库），并支持筛选、路线高亮和时间线浏览。

## 功能

- 地图标注上海红色景点，点击查看简介
- 免费高清底图（默认高清街道图，可切换标准街道图）
- 地图加入“上海边界遮罩 + 硬限制”，拖拽/缩放不会离开上海
- 点击任意景点可联动显示该地点场景：优先全景，失败自动回退到 3D 地图
- 3D 视图按景点馆址坐标校准并使用独立相机参数，定位更贴近真实馆址
- 点击景点时会在 ArcGIS 建筑图层中锁定最近建筑并高亮，3D 观察目标更准确
- 关键词搜索（名称 / 简介 / 关键词）
- 按行政区、类型筛选
- 推荐路线一键高亮（建党初心线 / 抗战记忆线 / 城市人文线）
- 点击路线自动生成打卡顺序（到达/离开时间）与预计总用时
- 每段路线并排展示四种方式：`步行 / 骑行 / 地铁 / 驾车`
- 驾车时长优先使用高德实时路况（需配置 Key），失败自动回退模型估算
- 步行/骑行/地铁时长为稳定模型估算，并统一基于同一路段距离计算
- 键盘可访问：景点卡片/路线步骤支持 `Tab + Enter/Space`，海报弹窗支持 `Esc` 关闭与焦点循环
- 一键转发：系统分享或复制当前页面链接（含筛选/路线状态）
- 一键生成分享海报（内含二维码 + 路线摘要），适合发微信好友
- 每个标点展示具体地址与详细介绍
- 景点列表与地图联动定位
- 历史时间线自动排序展示
- 桌面端和移动端自适应

## 运行方式

1. 克隆仓库并进入目录：

```bash
git clone https://github.com/<你的GitHub用户名>/shanghai-red-landmarks-app.git
cd shanghai-red-landmarks-app
```

2. 启动本地静态服务（推荐）：

```bash
python3 -m http.server 5173
```

3. 浏览器打开：

```text
http://localhost:5173
```

4. （可选）配置实时路况/街景 Key：

```bash
cp config/runtime-config.example.js config/runtime-config.js
```

然后编辑 `config/runtime-config.js`：

```js
window.RUNTIME_CONFIG = {
  amapKey: "你的高德 Web Service Key",
  baiduAk: "你的百度街景 AK"
};
```

说明：Key 走前端公开模式，请务必在对应平台设置域名白名单。

也可以直接双击 `index.html` 打开，但地图瓦片在某些浏览器设置下可能受限制，建议使用本地服务。

## 键盘操作说明

- 使用 `Tab / Shift + Tab` 在筛选项、景点卡片、路线按钮和弹窗按钮之间切换焦点。
- 在景点卡片和路线步骤上按 `Enter` 或 `Space` 可触发地图定位。
- 海报弹窗打开后，焦点会自动进入弹窗；按 `Esc` 可关闭，关闭后焦点会返回触发按钮。

## 路线缓存策略

- 路线分段结果会写入 `localStorage`，键名：`shanghai-red-landmarks-drive-traffic-v1`。
- 缓存按路段键（`fromId->toId`）存储，字段包含：`distanceMeters`、`walkMinutes`、`bikeMinutes`、`metroMinutes`、`driveMinutes`、`driveTrafficLevel`、`updatedAt`、`expiresAt`。
- 驾车实时路况缓存 TTL 为 `5 分钟`；步行/骑行/地铁模型缓存 TTL 为 `24 小时`。
- 最大缓存条目数为 300，超出后自动按最近更新时间清理。
- 若浏览器禁用或限制 `localStorage`（如隐私模式），会自动降级为仅内存缓存，不影响页面使用。
- 场景模式偏好会写入 `shanghai-red-landmarks-panorama-pref-v1`。

## 性能说明

- 搜索输入使用 `180ms` 防抖，减少高频重绘。
- 地图标点采用按 `spotId` 的差量更新，避免每次筛选全量重建 marker。
- 路线路网校正数据回填后仅刷新路线相关区域（路线卡片/计划/提示/里程），不触发全页重绘。
- 上海边界使用简化 GeoJSON，兼顾边界精度与移动端性能。

## 准确性体检脚本

执行：

```bash
node scripts/audit-accuracy.js
```

输出：

- 报告文件：`reports/accuracy-audit.md`
- 固定字段：`spotId`、`coordStatus`、`boundaryStatus`、`routeLegStatus`、`driveTrafficSource`、`panoramaSource`、`fallbackReason`

脚本会在有 Key 时统计实时路况命中率；无 Key 时会明确标记降级状态。

## 交通估时参数面板（实验）

- 在筛选区展开 `交通估时参数（实验）` 可调节步行/地铁/打车等模型参数。
- 参数变更后会自动重算路线时长并刷新路线区块，无需刷新页面。
- 参数会保存到当前浏览器本地（`localStorage`），下次打开会自动恢复。
- 点击 `恢复默认参数` 可一键回到默认模型。

默认参数：

- 步行速度 `4.8 km/h`，步行缓冲 `2 分钟`
- 地铁速度 `26 km/h`，地铁缓冲 `14 分钟`
- 地铁+步行速度 `23 km/h`，地铁+步行缓冲 `16 分钟`
- 打车速度 `30 km/h`，打车缓冲 `10 分钟`
- 打车引擎放大系数 `1.35`，打车额外分钟 `4`

## 分享说明

- 点击页面上的 `转发给朋友` 可直接调用系统分享（支持时）或自动复制链接。
- 点击 `生成分享海报` 可导出二维码海报，适合微信图片分享。
- 链接会包含当前筛选、选中路线和地图视角，朋友打开后可还原同一视图。
- URL 参数保持向后兼容，新增可选参数 `sceneMode=panorama|scene3d`（缺省为 `panorama`）。
- 如果你分享的是 `localhost` 链接，朋友通常无法直接访问；需要把项目部署到可公网访问地址后再分享。

## GitHub Pages 部署（推荐稳定分享）

项目已内置 GitHub Pages 自动发布工作流：`.github/workflows/deploy-pages.yml`

1. 在 GitHub 新建一个空仓库（不要勾选初始化 README），例如 `shanghai-red-landmarks-app`。
2. 在本地执行：

```bash
cd shanghai-red-landmarks-app
./publish-github-pages.sh https://github.com/<你的GitHub用户名>/<仓库名>.git
```

3. 首次发布后，到仓库 `Settings -> Pages` 检查 Source 是否为 `GitHub Actions`。
4. 等待 Actions 运行完成后，访问：

```text
https://<你的GitHub用户名>.github.io/<仓库名>/
```
